# Obsidian Usage Guide

## ใช้โฟลเดอร์ไหนเก็บอะไร
- `00 Core` = กฎหลัก ตัวตน วิธีทำงานที่ต้องยึดตลอด
- `00 Inbox` = กล่องพักโน้ตชั่วคราว ยังไม่จัดหมวด
- `01 Daily` = บันทึกรายวัน
- `10 Projects` = งานที่มีจุดจบชัด
- `20 Areas` = เรื่องที่ต้องดูแลต่อเนื่อง
- `30 Resources` = ความรู้และข้อมูลอ้างอิง
- `50 MOCs` = หน้าสารบัญหรือหน้ารวมลิงก์
- `80 Archive` = ของเก่า เก็บไว้เฉย ๆ
- `90 Templates` = แม่แบบสำหรับสร้างโน้ตใหม่

## กติกาง่าย ๆ
- ถ้าเป็น **ความรู้ใช้อ้างอิง** → ใส่ `30 Resources`
- ถ้าเป็น **เรื่องที่ต้องดูแลต่อเนื่อง** → ใส่ `20 Areas`
- ถ้าเป็น **งานมีเป้าหมายและจบได้** → ใส่ `10 Projects`
- ถ้าเป็น **โน้ตชั่วคราว** → โยนเข้า `00 Inbox` ก่อน
- ถ้าเป็น **หน้ารวมทางลัด** → ทำเป็น `MOC`

## โน้ตสำคัญของ Hermes
- `00 Core/Hermes Identity and Operating Rules.md` = ตัวตนและกฎหลัก
- `00 Core/AI Context Bridge.md` = วิธีรีแอนเคอร์บริบทตอนสลับโมเดล
- `00 Core/START HERE - Hermes Recovery.md` = จุดเริ่มกู้บริบท
- `00 Core/Latest Work Checkpoint.md` = งานล่าสุดที่กำลังทำ
- `20 Areas/Hermes/Hermes Operating Notes.md` = โน้ตปฏิบัติการประจำวัน

## ใช้งานยังไงให้ไม่รก
- เขียนสั้น ๆ ให้จับใจความได้เร็ว
- แยกโน้ตตามหน้าที่ ไม่ยัดทุกอย่างไว้ไฟล์เดียว
- ถ้าโน้ตเริ่มยาวและมีหลายเรื่อง ให้แยกเป็นไฟล์ย่อย
- ถ้ามีหัวข้อเกี่ยวกันหลายอัน ให้ทำลิงก์กลับไปหา note หลัก

## จำง่ายมาก
- **Inbox** = รอจัด
- **Projects** = งานมีจบ
- **Areas** = เรื่องดูแลยาว
- **Resources** = ความรู้
- **MOCs** = สารบัญ
- **Archive** = ของเก่า
- **Templates** = แม่แบบ
