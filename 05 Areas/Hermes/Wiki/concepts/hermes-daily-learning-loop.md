# Daily Learning Loop

## คืออะไร
Post-task reflection pattern ของเฮอ ใช้หลังจบงานสำคัญ หรืองานที่มี bug/error/workflow ใหม่ เพื่อตกผลึกบทเรียนและป้องกันพลาดซ้ำ

## 5 คำถามทุกครั้งหลังจบงาน
1. **ทำอะไรไปบ้าง** — สรุปงานที่ทำ
2. **ปัญหา/bug/error ที่เจอ** — อะไรพัง อะไรไม่เป็นไปตามแผน
3. **วิธีแก้ที่ใช้ได้จริง** — อะไรที่แก้แล้วเวิร์ค
4. **ควรบันทึกเป็นอะไร** — Memory / Skill / Workflow / Bug Note / Best Practice
5. **วิธีปรับปรุงครั้งหน้า** — ถ้าทำอีกจะทำต่างยังไง

## กฎ
- ถ้ามีขั้นตอนใช้ซ้ำได้ → เสนอสร้าง reusable skill ทันที
- ถ้าเป็น bug ที่มีโอกาสเจอซ้ำ → เสนอสร้าง bug note
- ถ้าเป็น workflow สำคัญ → เสนอบันทึกเป็น workflow
- ไม่ต้องอธิบายยาว เน้น actionable
